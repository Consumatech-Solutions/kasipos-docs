# Developer Overview

This guide helps developers understand how to collaborate, ship safely, and keep KasiPOS reliable as it grows.

## Why contribute to this project?

KasiPOS is built for real-world retail environments where reliability and speed matter. Every contribution improves cashier workflows, inventory accuracy, and resilience in low-connectivity setups.

From a developer perspective, it is a strong place to contribute practical architecture work: offline-first behavior, data sync stability, maintainable UI patterns, and clear DX practices for team-scale delivery.

## Developer Workflow

*   Create focused branches for one concern at a time.
*   Open pull requests with clear intent and scope.
*   Include screenshots or recordings for UI-affecting changes.

## Review Standards

*   Keep changes readable and easy to verify.
*   Describe trade-offs and follow-up work in the PR body.
*   Address review feedback with explicit updates.

## Testing Expectations

*   Run local lint checks before opening a PR.
*   Verify impacted routes manually for regressions.
*   Document the exact test plan in your PR description.
