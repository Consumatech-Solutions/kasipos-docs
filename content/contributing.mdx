# Contributing

Follow coding standards, branch strategy, and review practices when contributing.

## Workflow

*   Create focused branches for one concern at a time.
*   Open pull requests with clear intent and scope.
*   Include screenshots or recordings for UI-affecting changes.

## Review Standards

*   Keep changes readable and easy to verify.
*   Describe trade-offs and follow-up work in the PR body.
*   Address review feedback with explicit updates.

## Testing Expectations

*   Run local lint checks before opening a PR.
*   Verify impacted routes manually for regressions.
*   Document the exact test plan in your PR description.
