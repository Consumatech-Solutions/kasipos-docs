# Contributing

This guide defines the standard contribution workflow for the project.
The goal is to keep collaboration consistent, reviews efficient, and code quality maintainable.

## Workflow
### Fork Repository

Start by forking the repository to your personal GitHub account.
This allows you to work independently without affecting the main project repository.

*   Create focused branches for one concern at a time.
*   Open pull requests with clear intent and scope.
*   Include screenshots or recordings for UI-affecting changes.

### Pourquoi ?

- Travailler sans affecter directement le dépôt principal
- Isoler vos changements
- Faciliter la collaboration

### Commande :
```git clone https://github.com/<your-username>/<repository-name>.git```

## Review Standards


*   Keep changes readable and easy to verify.
*   Describe trade-offs and follow-up work in the PR body.
*   Address review feedback with explicit updates.

## Testing Expectations

*   Run local lint checks before opening a PR.
*   Verify impacted routes manually for regressions.
*   Document the exact test plan in your PR description.
